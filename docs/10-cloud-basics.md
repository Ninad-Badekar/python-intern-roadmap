# Module 4: Cloud Basics

**Goal:** Understand core cloud concepts and deploy a simple Python app (and store data/models) on one cloud provider.

**Duration:** 1–2 weeks

**Prerequisites:** [Module 3 — SQL Basics](./05-sql-basics.md); Python basics. Flask/FastAPI help for the deploy project later.

> Pick **one** cloud for hands-on: **AWS**, **GCP**, or **Azure**. Concepts transfer across providers.

---

## Topics & checklist

### Cloud fundamentals
- [ ] Why cloud? elasticity, pay-as-you-go, managed services
- [ ] IaaS vs PaaS vs SaaS
- [ ] Regions and availability zones
- [ ] Shared responsibility model (high level)

### Core services
- [ ] Compute: VMs / instances / App Service / Cloud Run
- [ ] Object storage (S3 / GCS / Blob)
- [ ] Managed databases (RDS / Cloud SQL / Azure Database)
- [ ] IAM: users, roles, policies (least privilege)
- [ ] Networking intro: VPC, public/private, firewalls/security groups

### Deploying Python workloads
- [ ] Environment variables and secrets
- [ ] Deploy Flask or FastAPI (PaaS or container)
- [ ] Store datasets/models in object storage
- [ ] Docker basics: image, container, Dockerfile
- [ ] CI/CD awareness with GitHub Actions

---

## Mini project

**Deploy one app**
- [ ] Public HTTPS endpoint for a small API or web app
- [ ] README with architecture + deploy steps
- [ ] No secrets in Git
